## Description

Implementar design responsivo mobile-first para o Health Dashboard, garantindo usabilidade em dispositivos móveis.

### Problem Statement

O dashboard não era otimizado para telas pequenas, com elementos cortados, touch targets pequenos e navegação difícil em mobile.

## Acceptance Criteria

```gherkin
Feature: Mobile Responsiveness

Scenario: Touch targets adequados
  Given estou em um dispositivo touch
  When toco em botões e links
  Then todos devem ter área mínima de 44x44px

Scenario: Layout adaptativo
  Given acesso em tela < 768px
  When visualizo o dashboard
  Then o layout deve se adaptar para coluna única
  And tabelas devem ser scrolláveis horizontalmente

Scenario: Navegação mobile
  Given estou em mobile
  When uso o header
  Then a navegação deve estar acessível
  And não deve haver overflow horizontal
```

## Dev Notes

### Breakpoints

```css
sm: 640px
md: 768px
lg: 1024px
xl: 1280px
```

### Mobile-First Pattern

```css
/* Mobile default */
.element { /* mobile styles */ }

/* Desktop override */
@media (min-width: 768px) {
  .element { /* desktop styles */ }
}
```

### Touch Targets (WCAG 2.5.5)

```css
@media (max-width: 767px) {
  button, .btn, .nav-link, a {
    min-height: 44px;
    min-width: 44px;
  }
}
```
