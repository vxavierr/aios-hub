## Description

Mover os arquivos de mapping (scripts JS e resultados JSON) da raiz do HUB para um diretório organizado em `docs/mapping/`.

### Problem Statement

A raiz do HUB continha 15+ arquivos JavaScript de mapping e 10+ arquivos JSON de resultados, criando poluição visual e dificultando a navegação.

## Acceptance Criteria

```gherkin
Feature: Organizar arquivos de mapping

Scenario: Scripts JS movidos
  Given existem arquivos map-*.js na raiz
  When executo a reorganização
  Then todos os scripts de mapping devem estar em docs/mapping/
  And a raiz deve conter apenas package.json e package-lock.json

Scenario: Resultados JSON movidos
  Given existem arquivos *-mapping*.json na raiz
  When executo a reorganização
  Then todos os JSONs de mapping devem estar em docs/mapping/

Scenario: Funcionalidade preservada
  Given os scripts foram movidos
  When preciso executar um script
  Then o script deve funcionar a partir de docs/mapping/
```

## Scope

### In Scope

- Mover todos os scripts map-*.js para docs/mapping/
- Mover todos os JSONs de mapping para docs/mapping/
- Mover arquivos auxiliares (comet-scan.js, full-scan.js, etc.)
- Manter package.json e package-lock.json na raiz

### Out of Scope

- Refatorar os scripts
- Consolidar scripts duplicados
- Criar documentação dos scripts
