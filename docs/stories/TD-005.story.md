## Description

Expandir o sistema de design tokens do Health Dashboard para incluir tipografia, transições, z-index e outros tokens necessários, e atualizar todos os componentes para usar os tokens em vez de valores hardcoded.

### Problem Statement

O Health Dashboard tinha apenas design tokens básicos (colors, spacing, radius, shadows). Valores de tipografia, transições e outros estilos estavam hardcoded nos componentes CSS, dificultando a manutenção e consistência.

## Acceptance Criteria

```gherkin
Feature: Design Tokens Completos

Scenario: Tokens de tipografia
  Given o sistema precisa de consistência tipográfica
  When defino os tokens
  Then devem existir font-size, font-weight, line-height, letter-spacing
  And todos os componentes devem usar os tokens

Scenario: Tokens de transição
  Given os componentes têm animações
  When defino os tokens
  Then devem existir transições padronizadas (fast, normal, slow, slower)
  And transition-duration hardcoded deve ser substituído

Scenario: Tokens de componente
  Given ícones e spinners têm tamanhos fixos
  When defino os tokens
  Then devem existir tokens para icon-size e spinner-size
```

## Scope

### In Scope

- Expandir tokens em index.css com tipografia completa
- Adicionar tokens de transição
- Adicionar tokens de z-index
- Adicionar tokens de tamanho de componentes
- Atualizar todos os componentes CSS para usar tokens

### Out of Scope

- Implementar tema light/dark (TD-008)
- Criar documentação visual dos tokens
- Migrar para CSS-in-JS
